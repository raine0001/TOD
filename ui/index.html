TOD Console Help
Current ActionsAction Output shows live request id, execution status, gate and validator, plus comms state (active/frozen) from heartbeat age versus stall threshold.
System PostureCounters are live workload pressure: goals, active executions, pending confirmations, blocked items, capabilities, and executor health. All zeros often means steady idle between handoffs.
Top Probe BarMIM probe reflects ping reachability only. TOD Active reflects local UI/API activity. They can differ and both may still be healthy.
Action Output + PendingTimeline is newest-first. Pending queue predicts upcoming objective task ids and shrinks as completions are recorded.
Shared State BusRefresh reads a snapshot only. It does not write commands, but can be heavy during active execution windows.
Scorecard TrendSparkline is oldest to latest scorecard samples from the engineering loop; direction summarizes improving/flat/declining trend.
Engineering LoopShows run/cycle state, maturity band, approval pressure, penalties, and phase trends that affect delivery velocity.
MIM Call TelemetryTime is local timestamp when request completed. Latency ms is round-trip request duration.
Raw Log StreamRendered newest-first for fast triage; table and execution events summarize the same telemetry window.
Quick Refresh ReliabilitySafe refresh mode now reloads status panels without dispatching a heavy TOD action run.
Share ArtifactsOpen uses inline preview endpoint; Download keeps attachment behavior. Refresh updates available artifacts and timestamps.
Executive SummaryNarrative under Current Project Marker includes objective, percent complete, tasks remaining, ETA estimate, purpose text, and health signal.
